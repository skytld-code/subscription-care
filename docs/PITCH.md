# 발표 피치 & Q&A — 구독챙김 (2026-07-10)

## 0. 한 줄 포지셔닝

> **"열어야 보이는 앱이 아니라, 찾아오는 앱."**
> 경쟁 발표(END.)가 나오면: "쟤는 다 보여주는 앱, 저는 결정하게 하는 앱입니다."

---

## 1. 발표 스크립트 (약 3분)

**[문제 — 30초]**
"구독 해지를 못 하는 이유는 의지가 없어서가 아닙니다. '해지해야겠다'고 생각나는 시점과 실제로 해지할 수 있는 시점이 다르기 때문이에요. 결제되고 나서야 알아차리고, 무료체험은 끝나고 나서야 기억나죠."

**[해법 — 30초]**
"구독챙김은 가계부가 아닙니다. 이름·금액·결제일, 3개 필드만 30초 입력하면 그다음부터는 앱을 열 일이 없어요. 결정해야 하는 순간에 잠금화면으로 찾아갑니다."

**[시연 — 90초]** (동선은 §2)

**[수익성 — 30초]**
"서버가 없어서 운영비가 0원입니다. 그래서 월 구독료 대신 6,900원 평생 소장이 가능해요. 구독을 줄여주는 앱이 구독료를 받을 순 없으니까요. 페이월은 사용자가 가치를 5번 경험한 뒤, 6번째 등록 순간에 뜹니다."

## 2. 시연 동선 (리허설 필수)

1. 랜딩 히어로의 폰 **탭** → 잠금화면에서 앱이 떠오르는 연출 (후킹)
2. 나타나는 **"지금 이 앱, 크게 보기 ↗"** 클릭 → 새 탭 전체 화면
3. 홈 상단 **"어제 유튜브 프리미엄 14,900원이 결제됐어요"** 카드 → "금액이 달랐어요" → 16,900 입력 → **가격 인상 감지 배너** (월 +2,000원 · 연 +24,000원)
4. 보라색 **구독 ROI 카드** — "밀리의 서재, 이번 달 1시간 · 시간당 9,900원" → 해지 알아보기 → **실제 해지 검색 새 탭** (탭만 보여주고 닫기)
5. 디즈니+ (체험 D-3) 행 탭 → **알림 미리보기** → 배너의 `계속 쓸게요 / 해지 알아보기` 버튼 시연
6. 추가 탭에서 6번째 구독 등록 → **페이월** → "결제하고 계속하기" → 등록 완료 배너

⚠️ 6번째 등록에서 페이월이 뜨는 건 **의도된 연출** — 당황하지 말고 수익 모델 설명으로 연결할 것.

## 3. 예상 Q&A

**Q. 왜 서버가 없나요? / 기술이 부족해서 아닌가요?**
"제품 선택입니다. 다루는 데이터가 구독 이름·금액·결제일 3개 필드뿐이라 서버에 모을 게 없어요. 알림도 iOS 로컬 알림으로 폰이 스스로 예약합니다. 이 선택으로 얻는 게 세 가지예요 — 금융 데이터를 아예 안 가지니 개인정보 이슈가 원천적으로 없고, 운영비 0원이라 단건 결제가 성립하고, 비행기 모드에서도 돌아갑니다."

**Q. 폰 바꾸면 데이터 날아가요?**
"iCloud 기기 백업에 포함돼서 따라옵니다. 다기기 동기화가 필요해지면 iCloud 동기화를 붙일 건데, 그것도 애플 인프라라 여전히 저희 서버는 없어요."

**Q. 경쟁 서비스(END.)가 기능이 더 많던데요?**
"맞아요, 그리고 그게 그쪽의 약점이라고 봅니다. 기능이 많다는 건 사용자가 입력할 것도, 앱을 열어야 할 이유도 많다는 뜻이에요. 저희는 반대로 겁니다 — 앱을 안 열수록 잘 작동하는 제품이요. 실제로 저희 KPI에 DAU가 없습니다. 핵심 지표는 '알림에서 버튼을 누른 비율'이에요."

**Q. 스크린타임 연동, 진짜 가능해요?**
"애플이 원시 사용시간 데이터를 자유롭게 주진 않아요. 대신 DeviceActivityMonitor의 임계치 이벤트로 '이번 달 이 앱을 N시간 이상 썼는지'는 온디바이스로 판정할 수 있습니다. '2시간도 안 썼네요' 수준의 판정이면 ROI 알림에 충분하고, 서버 없이 됩니다. 웹 서비스는 스크린타임 접근 자체가 불가능하다는 점이 저희 차별화고요. Family Controls 승인이 필요해서 로드맵 리스크로 관리 중입니다."

**Q. 해지 페이지 링크는 어떻게 유지해요?**
"주요 서비스는 URL 맵으로 직접 연결하고, 없는 서비스는 '서비스명 + 해지' 검색으로 폴백합니다. 링크가 깨져도 사용자가 길을 잃지 않게요."

**Q. 무료 사용자한테 뭘로 돈 벌어요?**
"무료 5개 한도 → 6,900원 평생 소장 단건 결제입니다. 서버비가 없어서 이 모델이 성립해요. 구독 관리 앱이 월 구독료를 받는 것 자체가 브랜드 모순이라고 판단했습니다."

**Q. 왜 iOS만?**
"잠금화면 알림·뱃지·스크린타임 등 이 제품의 핵심 표면이 전부 OS 기능이에요. 한 플랫폼에서 push 경험을 완성하는 게 두 플랫폼에서 어중간한 것보다 낫다고 봤습니다. 안드로이드는 검증 후 확장입니다."
