# 발표 피치 & Q&A — 구독챙김 (2026-07-10)

## 0. 한 줄 포지셔닝

> **"열어야 보이는 앱이 아니라, 찾아오는 앱."**
> 경쟁 발표(END.)가 나오면: "쟤는 다 보여주는 앱, 저는 결정하게 하는 앱입니다."

---

## 1. 발표 스크립트 (약 3분)

**[문제 — 45초]**
"구독 해지를 못 하는 이유는 의지가 없어서가 아닙니다. '해지해야겠다'고 생각나는 시점과 실제로 해지할 수 있는 시점이 다르기 때문이에요. 결제되고 나서야 알아차리고, 무료체험은 끝나고 나서야 기억나죠.

제 느낌이 아니라 소비자원 통계입니다. 무료체험 후 자동결제 피해 신청이 2022년 26건에서 2024년 71건, 3년 새 3배 가까이 늘었어요. 피해 1위가 '유료 전환 고지 미흡', 2위가 '무료기간 내 해지 방해'입니다. 그런데 돈을 전액 돌려받은 사람은 41.7%뿐이에요. **사후 구제가 안 되니, 결제 전에 아는 것만이 답입니다.**"

**[해법 — 30초]**
"구독챙김은 가계부가 아닙니다. 이름·금액·결제일, 3개 필드만 30초 입력하면 그다음부터는 앱을 열 일이 없어요. 결정해야 하는 순간에 잠금화면으로 찾아갑니다."

**[시연 — 90초]** (동선은 §2)

**[수익성 — 30초]**
"서버가 없어서 운영비가 0원입니다. 그래서 월 구독료 대신 6,900원 평생 소장이 가능해요. 구독을 줄여주는 앱이 구독료를 받을 순 없으니까요. 페이월은 사용자가 가치를 5번 경험한 뒤, 6번째 등록 순간에 뜹니다."

## 2. 시연 동선 (리허설 필수)

**랜딩 스크롤이 곧 발표 목차**: 히어로(후킹) → 01 문제 → 02 차별화(4개 기능 카드) → 03 작동 방식 → 04 미리보기(폰 3대: 등록→뱃지→알림 순) → 05 가격 → 06 설치. 상단 내비게이션 메뉴로 섹션 점프 가능.

1. 랜딩 히어로의 폰 **탭** → 잠금화면에서 앱이 떠오르는 연출 (후킹)
2. 나타나는 **"지금 이 앱, 크게 보기 ↗"** 클릭 → 새 탭 전체 화면
3. 홈 상단 **"어제 유튜브 프리미엄 14,900원이 결제됐어요"** 카드 → "금액이 달랐어요" → 16,900 입력 → **가격 인상 감지 배너** (월 +2,000원 · 연 +24,000원)
4. 보라색 **구독 ROI 카드** — "밀리의 서재, 이번 달 1시간 · 시간당 9,900원" → 해지 알아보기 → **실제 해지 검색 새 탭** (탭만 보여주고 닫기)
5. 디즈니+ (체험 D-3) 행 탭 → **알림 미리보기** → 배너의 `계속 쓸게요 / 해지 알아보기` 버튼 시연
6. 추가 탭에서 6번째 구독 등록 → **페이월** → "결제하고 계속하기" → 등록 완료 배너

⚠️ 6번째 등록에서 페이월이 뜨는 건 **의도된 연출** — 당황하지 말고 수익 모델 설명으로 연결할 것.

⚠️ **발표 전 실행 방법**: 파일을 더블클릭(file://)하지 말고, 프로젝트 폴더에서 `python3 -m http.server 8000` 실행 후 `http://localhost:8000` 으로 열 것. 미리보기 폰(실제 앱)에서 등록 → 뱃지·배너 연동이 localStorage 이벤트 기반이라 서버로 열어야 모든 브라우저에서 확실히 동작함. 시연 직전 앱 설정에서 '데이터 초기화'로 샘플 상태 복원 가능(초기화 후 새로고침하면 샘플 5개 재생성).

## 3. 예상 Q&A

**Q. 왜 서버가 없나요? / 기술이 부족해서 아닌가요?**
"제품 선택입니다. 다루는 데이터가 구독 이름·금액·결제일 3개 필드뿐이라 서버에 모을 게 없어요. 알림도 iOS 로컬 알림으로 폰이 스스로 예약합니다. 이 선택으로 얻는 게 세 가지예요 — 금융 데이터를 아예 안 가지니 개인정보 이슈가 원천적으로 없고, 운영비 0원이라 단건 결제가 성립하고, 비행기 모드에서도 돌아갑니다."

**Q. 폰 바꾸면 데이터 날아가요?**
"iCloud 기기 백업에 포함돼서 따라옵니다. 다기기 동기화가 필요해지면 iCloud 동기화를 붙일 건데, 그것도 애플 인프라라 여전히 저희 서버는 없어요."

**Q. 경쟁 서비스(END.)가 기능이 더 많던데요?**
"맞아요, 그리고 그게 그쪽의 약점이라고 봅니다. 기능이 많다는 건 사용자가 입력할 것도, 앱을 열어야 할 이유도 많다는 뜻이에요. 저희는 반대로 겁니다 — 앱을 안 열수록 잘 작동하는 제품이요. 실제로 저희 KPI에 DAU가 없습니다. 핵심 지표는 '알림에서 버튼을 누른 비율'이에요."

**Q. 스크린타임 연동, 진짜 가능해요?**
"애플이 원시 사용시간 데이터를 자유롭게 주진 않아요. 대신 DeviceActivityMonitor의 임계치 이벤트로 '이번 달 이 앱을 N시간 이상 썼는지'는 온디바이스로 판정할 수 있습니다. '2시간도 안 썼네요' 수준의 판정이면 ROI 알림에 충분하고, 서버 없이 됩니다. 웹 서비스는 스크린타임 접근 자체가 불가능하다는 점이 저희 차별화고요. Family Controls 승인이 필요해서 로드맵 리스크로 관리 중입니다."

**Q. 가격 인상을 '자동으로' 추적할 수는 없어요? 지금은 사용자가 입력하잖아요.**
"완전 자동은 내 계좌·카드 내역을 읽어야 하는데, 그 순간 마이데이터 연동·본인인증·서버가 전부 따라오고 저희 철학과 정반대가 됩니다. 그래서 두 축으로 갑니다. 첫째, 지금의 결제 확인 카드 — 사용자가 탭 한 번으로 확정해주는 방식이라 요금제가 제각각이어도 정확해요. 둘째, 로드맵에 있는 공시가 카탈로그 — 주요 서비스의 공식 가격표를 정적 파일로 앱에 배포해서, 넷플릭스가 가격 인상을 공지하면 '공식 가격이 올랐어요, 내 요금도 확인해보세요'라고 선제적으로 알려줍니다. 정적 파일이라 서버 없이 되고, 가격 변경은 1년에 몇 번 수준이라 유지보수도 가능합니다."

**Q. 해지 페이지 링크는 어떻게 유지해요?**
"주요 서비스는 URL 맵으로 직접 연결하고, 없는 서비스는 '서비스명 + 해지' 검색으로 폴백합니다. 링크가 깨져도 사용자가 길을 잃지 않게요."

**Q. 6,900원 결제는 어떻게 받아요? 계정도 없다면서요?**
"애플 인앱결제(StoreKit)의 '비소모성 상품'으로 팝니다. 사용자는 Face ID 한 번이면 끝이고, 구매 기록은 사용자의 Apple ID에 묶여요. 그래서 저희가 계정이나 결제 서버를 만들 필요가 없고, 기기를 바꿔도 '구매 복원' 버튼으로 되살아납니다 — 서버 없는 구조와 정확히 맞물리는 결제 방식이에요. 수수료는 애플 중소개발자 프로그램 기준 15%입니다."

**Q. 무료 사용자한테 뭘로 돈 벌어요?**
"무료 5개 한도 → 6,900원 평생 소장 단건 결제입니다. 서버비가 없어서 이 모델이 성립해요. 구독 관리 앱이 월 구독료를 받는 것 자체가 브랜드 모순이라고 판단했습니다."

**Q. 왜 '서버가 없으면 단건 결제가 가능'한 거예요? (경제 논리 3단계)**
1. "앱이 월 구독료를 받는 이유는, 사용자가 쓰는 동안 개발자에게 비용이 계속 나가서예요. 서버 저장 비용, 푸시를 쏘는 서버, 계정·개인정보 관리 — 사용자 1명이 1년 더 쓰면 1년치 비용이 더 나갑니다. 그래서 매달 받아야 하고, 이런 구조에서 '평생권'을 팔면 오래 쓰는 유저마다 적자가 쌓여요."
2. "구독챙김은 그 세 가지가 전부 다른 주체에게 가 있어요. 데이터는 사용자 폰에, 알림은 iOS 로컬 알림이라 폰이 알람시계처럼 스스로 울리고, 계정·결제는 Apple ID가 관리합니다. 사용자가 1명이든 10만 명이든 제게 발생하는 유지 비용이 0원이에요. 고정비는 애플 개발자 계정 연 $99가 전부입니다."
3. "월 구독료는 매달 나가는 비용을 회수하는 수단인데, 나가는 비용이 없으니 매달 받을 이유가 없어요. 6,900원 한 번이면 그 사용자에 대한 원가는 영구 회수 완료입니다."

한 줄 버전: **"월 구독료는 서버비 고지서를 사용자에게 나눠 내게 하는 겁니다. 저희는 고지서 자체가 없어서, 나눠 낼 것도 없어요."**

꼬리 질문 대비:
- "업데이트 개발비는요?" → "개발비는 신규 판매로 회수합니다. 핵심은 기존 사용자 유지 비용이 0원이라는 것."
- "iCloud 동기화 붙이면 서버비 생기죠?" → "CloudKit은 데이터가 사용자 본인의 iCloud 용량에 저장돼요. 그때도 개발자 부담 0원입니다."

**Q. 왜 iOS만?**
"잠금화면 알림·뱃지·스크린타임 등 이 제품의 핵심 표면이 전부 OS 기능이에요. 한 플랫폼에서 push 경험을 완성하는 게 두 플랫폼에서 어중간한 것보다 낫다고 봤습니다. 안드로이드는 검증 후 확장입니다."
